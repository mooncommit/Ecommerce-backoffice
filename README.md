## GitHub 협업 규칙

- `main` 브랜치에 직접 push 하지 않습니다.
- 각자 브랜치를 생성하여 작업합니다.
- 작업 완료 후 `Pull Request`로 병합합니다.
- 작업 전 최신 `main` 브랜치를 pull 받습니다.

## 커밋 타입

- `feat`: 새로운 기능 추가
- `fix`: 버그 수정
- `refactor`: 프로덕션 코드 리팩토링

## 커밋 메시지 규칙

커밋 메시지는 아래 형식으로 작성합니다.

```bash
커밋타입 : 제목

본문

```

## 커밋 메시지 규칙

- 커밋은 기능 단위로 나누어 작업합니다.
- 의미 없는 단순한 작업은 커밋을 지양합니다.

## 클래스 작명 규칙

클래스 작명은 아래 형식으로 작성합니다.

- 도메인 + 액션 + 요청/응답 + Dto
